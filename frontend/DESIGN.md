# ContestMind Design System

## 1. Design Theme

Modern competitive-programming + AI mentor aesthetic.

NOT:
* neon hacker
* crypto dashboard
* generic AI SaaS
* overdone glassmorphism

Wanted vibe:
* clean
* premium
* sharp
* fast
* focused
* technical
* minimal but intelligent

Reference direction:
* Linear
* Vercel
* Raycast
* modern IDE aesthetics
* ICPC/Codeforces competitive vibe

Theme details:
* dark-first UI
* slightly warm dark background (not pure black)
* subtle gradients only
* clean typography
* dense but readable layouts
* card-based analytics sections
* smooth hover animations
* modern charts
* strong spacing consistency
* no flashy animations
* no giant rounded blobs
* avoid “AI generated landing page” look

Primary accent:
* emerald / cyan / indigo mix
* use accent sparingly
* difficulty colors:
  * green → easy
  * blue → medium
  * orange → hard
  * red → expert

Visual feeling:
“Elite competitive programming training platform.”

Key UI goals:
* feels fast
* information-rich
* clean dashboards
* modern analytics
* strong developer tooling vibe
* personalized coaching system feel

## 2. Component Library

Use:
* TailwindCSS v4
* shadcn/ui components
* lucide-react icons

DO NOT generate:
* raw messy Tailwind everywhere
* giant monolithic JSX files
* inline styles
* overly complex animation systems

Preferred stack:
* Next.js App Router
* Tailwind v4
* shadcn/ui
* Framer Motion only where useful
* Recharts for analytics

Important:
Components should look handcrafted and production-grade,
NOT autogenerated.

## 3. UX Priorities

The frontend should heavily focus on:
* personalized dashboard
* weakness analytics
* recommendation cards
* contest performance insights
* progressive hints UI
* editorial simplification view
* semantic search experience
* solve probability visualization
* adaptive coaching feel

The platform should feel like:
“LeetCode + Linear + AI Coach for competitive programming.”

## 4. Frontend Engineering Rules

* modular components
* reusable cards
* responsive layouts
* avoid overengineering
* avoid massive state complexity initially
* prioritize clean architecture
* TypeScript strict mode
* loading skeletons
* empty states
* proper error states
* optimistic UI where useful

## 5. VERY IMPORTANT

Avoid:
* generic AI gradients everywhere
* chatbot-only design
* childish gamification
* generic startup hero sections
* excessive glassmorphism
* template-looking UI

The UI should feel:
credible,
serious,
technical,
premium,
and made for advanced programmers.
